POST 01. get access token
{{base_url}}/omnic/api/token/generate-token

Theget access tokenendpoint is used to generate a token.

How to Get an Access Token

Authorization

  • No Auth

Required Inputs

  • userName: The user’s username.

  • password: The user’s password.

Response Body

You will receive a JSON Web Token in the response:
“{{vault:json-web-token}}”

Set the Token for Authorization

access_token must be passed to all endpoints.
Example:
bearer-token={{vault:json-web-token}}

    "password": "String"

POST 02 buy insurance
{{base_url}}/omnic/api/insuranceService/insurance/policy/new

buy insurance endpoint is used to buy a new policy.

Buy Policy:

Get Available Insurance Options
Use the “get subrisks” endpoint to retrieve insurance options based on the “Insurance Class” you need.
Select Insurance Class
Choose an insurance class, for example:insuranceClass: "PRIVATE COMPREHENSIVE"
Send the Request
Include the selected insurance class and other required details in the payload to proceed with the purchase.

Validation Rules for Input:

  • Date: The date should be in day/month/year format (e.g., 12/01/2025).

  • Car Registration No.: The car registration number must not be greater than 11 characters.

  • Chasis Number: The chasis number must not contain the letters “I” and “O”.

The length of the chasis number must not be more than 17 characters.

  • Sum Insured Type: The sum insured should be numeric (i.e., a number)
